# 02 — Dataset Pipeline Specification

## Datasets

| Dataset | Role | Size used | Source |
|---|---|---|---|
| LIDC-IDRI | Training (VAE + diffusion) | 1,010 CT volumes | [PAPER] ref [1], TCIA public download |
| NLST | Inference only — unseen anatomical masks, generalization test | 881 volumes with ≥1 nodule annotation | [PAPER] ref [20]; nodule masks generated by an "ad-hoc U-Net" since NLST doesn't ship them natively |

**[PAPER]**: "LIDC-IDRI was used for training, while the NLST subset provided unseen anatomical
masks for inference." — i.e. NLST is never used for training, only to test how well the model
generalizes to lung/nodule shapes it never saw.

## Annotation source per dataset

- **LIDC-IDRI**: nodule masks + texture ratings (1–5) come directly from four radiologists'
  original annotations, parsed via `pylidc`. This is the authoritative source for the
  `nodule_texture_labels` used everywhere in this project.
- **NLST**: nodule bounding boxes only (ref [18], Sybil paper) — LAND's authors generated actual
  nodule *masks* from these boxes using a separately-trained U-Net (not released). We do **not**
  have this U-Net. **[OPEN QUESTION]** — for our NLST inference tests we'll need either (a) a
  bounding-box-to-mask heuristic, or (b) skip NLST entirely and validate generalization by
  held-out LIDC-IDRI patients instead. Current plan: (b), since it avoids depending on an
  unreleased model, and a LIDC-IDRI held-out split still validates real generalization.

## Lung segmentation

**[PAPER]**: "Lung regions in both datasets were segmented with a pre-trained open-source U-Net
[13]." Ref [13] = Hofmanninger et al., "Automatic lung segmentation in routine imaging..." — this
is exactly the model shipped in the `lungmask` PyPI package. We use `lungmask` directly rather
than retraining anything here — this is a faithful match, not an approximation.

## Sample construction

```
Healthy sample = CT volume + lung mask + all-zero nodule channel
Tumor sample   = CT volume + lung mask + real nodule mask (texture-scored)
```

Every LIDC-IDRI CT with at least one annotated nodule becomes a **tumor** training sample.
For **healthy** samples we have two options:
1. Use LIDC-IDRI scans that have zero annotated nodules (some exist in the dataset — patients
   scanned but cleared).
2. Take a tumor-bearing scan and mask out ("zero") its nodule channel, treating the same
   underlying CT as a healthy-conditioning example — but note the CT pixel data would still
   physically contain the nodule since we're not removing it from the image itself, just from
   the mask. This would teach the model an inconsistent mapping (mask says no nodule, but the
   CT ground truth it's learning from has one).

**Decision**: use option 1 only — true nodule-free LIDC-IDRI scans — for constructing empty-mask
training pairs, so mask and CT stay consistent. Cross-reference `06_open_questions.md`.

## Train/val/test split

**[INFERRED]** — paper doesn't specify a split ratio for LIDC-IDRI internally (only that all of
it is used for training, with NLST for external validation). We use 85/10/5 train/val/test,
split at the **patient** level (not scan/slice level) to avoid leakage, seed=42, stratified so
both healthy and tumor cases appear in all three splits.

## Output format

Preprocessed volumes + mask tensors are cached to disk as compressed `.npz` (float16 for the
CT volume, uint8 for the mask before its final [0,1] normalization) to avoid re-running DICOM
decoding, resampling, and mask inference on every training epoch. See `03_preprocessing.md` for
the exact preprocessing steps applied before caching.
